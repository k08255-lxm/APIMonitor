# API Monitor Android 小组件

此目录是一个独立的 Android 应用，为 API Monitor 服务提供原生 Material Design 3
仪表盘和可配置的主屏幕小组件。小组件使用 Java `RemoteViews`；应用仪表盘使用
AndroidX 和 Material Components。

## 功能

- 每个小组件实例都可以单独配置服务 URL 和可选的 Basic 密码。
- Basic 用户名固定为 `monitor`。
- 桌面组件目标尺寸为 `4 x 5`，会填满可用空间，并显示 Token、成本、请求、延迟、
  RPM、TPM、成功率、服务状态、活跃 Key、模型 TOP、累计总览和最近五次调用。
- 加载中、连接错误、最近调用为空和上次成功更新时间等状态会直接显示在小组件中。
- 刷新图标通过私有且未导出的 receiver 立即触发更新。小组件会在网络可用时由 WorkManager
  按 30 分钟最短周期自动刷新，安装或覆盖更新后会补一次刷新；Android 省电或厂商策略仍可能延后后台任务。
- 点击组件主体会打开原生仪表盘，不会跳转到浏览器。
- 原生仪表盘提供今日/24 小时/7 天范围、数据源选择、Token/请求趋势、核心指标、最近
  50 条调用、模型横向柱状排行、累计总览、下拉刷新和连接设置；小组件本身仍只显示五条。
- 趋势图可单独切换 6 小时、12 小时、24 小时或全部可用时间点；点按核心数据或模型图可在
  紧凑单位与精确数值之间切换，选择会保存在此设备。
- 原生仪表盘首页只显示后端摘要；“管理”会进入独立后端管理页，显示 Node.js 后端的地址、
  PID、实例 ID、启动时间和运行时长。配置面板密码后，可在此处重启、关闭服务，并管理
  Windows 登录后持续运行或随 cc-switch 启动两种开机自启模式，均使用无窗口启动器。
- 首页“检查更新”会读取 GitHub 的正式发布。检测到新 APK 后可一键下载、校验并打开系统
  安装器；首次需要在 Android 系统页面允许本 App 安装未知来源应用，授权后返回 App 会继续安装。
- 在 App 内点击“添加桌面小组件”时，首次会先填写服务地址和密码；后续添加的组件复用
  已验证的连接配置。直接从系统桌面添加时，没有已保存连接才会进入填写步骤；已有连接
  时会安全复用该连接。
- 移除小组件时，会从应用私有的 `SharedPreferences` 中删除其 URL、密码和更新元数据。

## 构建、安装与添加到桌面

这个目录是 Android 源码，不是已安装的手机应用。只有把 APK 安装到手机后，系统的
“小组件”列表才会出现“API 监测台”。

1. 在 Windows 安装 Android Studio，安装时选择 Android SDK Platform 35；然后用它打开
   `android-widget/` 目录。
2. 推荐在项目根目录双击 `构建安卓小组件.cmd`：它会先显示 Android SDK 的许可证确认，
   然后自动构建并验证已签名的 release APK：
   `app/build/outputs/apk/release/app-release.apk`。首次构建会保存本机私有签名信息，以便
   后续版本可以覆盖安装。已经运行过一键构建时，也可在 Android Studio 中选择
   **Build > Generate Signed Bundle / APK** 手动生成同一签名的 APK。
3. 把 `app-release.apk` 传到手机并安装。Android 首次安装时，需要为用于打开 APK 的应用
   授予“允许安装未知应用”的权限。
4. 在手机桌面长按空白处，进入“小组件”，找到“API 监测台”，拖到桌面。系统随即打开
   配置页面。
5. 填写 `http://<电脑局域网 IP>:8787`、密码 `DASHBOARD_PASSWORD`，先点“测试连接”，
   再点“保存并添加”。局域网 HTTP 会要求你确认一次明文传输提示。

添加后，点击桌面组件主体可打开完整的原生仪表盘，点击刷新图标可立即更新。打开
“API 监测台”应用可查看完整监控数据、修改服务地址和密码，再新增桌面组件。

APK 签名、私钥备份和“签名不一致”的一次性处理步骤见 [SIGNING.md](SIGNING.md)。

手机和电脑必须处于同一受信任 Wi-Fi，且 Windows 防火墙需要只在“专用网络”放行 TCP
`8787`。具体的服务启动和防火墙设置见项目根目录的 `README.md`。

## 配置与传输安全

输入监控服务地址，例如 `https://monitor.example.com:8787`。小组件会自动追加
`/api/dashboard?range=today`。如果监控服务启用了 `DASHBOARD_PASSWORD`，请在此处
输入相同的值；应用会使用用户名 `monitor` 通过 HTTP Basic 身份验证发送该密码。
仅当面板明确配置为无需身份验证时才留空。

强烈建议使用 HTTPS。仅针对受信任的私有局域网主机，配置界面也允许使用 `http://`
（localhost、`.local`、回环地址、链路本地地址或 RFC1918 IPv4 地址）。在明文 HTTP
上传输的 Basic 凭据，任何能够监听该局域网流量的人都可以看到。公共网络的明文地址
会被拒绝。应用不会安装宽松的 `TrustManager`，也不会关闭 TLS 证书或主机名校验。

URL 和密码会按小组件实例保存在应用私有存储中，不会写入日志或显示在小组件文字中。
网络响应大小限制为 512 KiB，并拒绝重定向，以避免把凭据发送到其他来源。

后端状态使用与监控面板相同的 Basic Auth。只有服务端已配置 `DASHBOARD_PASSWORD`，
App 才能使用用户名 `monitor` 和已保存的密码发送重启、关闭或开机自启管理请求；未配置时控制按钮会
保持禁用。开机自启只支持 Windows 当前用户：可以选择登录后持续运行，或只在检测到 cc-switch 启动后运行。
关闭后 HTTP API 无法自行启动，请在电脑端双击 `启动监控.cmd`，需要手机
访问时则双击 `启动手机监控.cmd`。Android App 不需要本机控制 token；绝不要把电脑上
`data/server-state.json` 的内容复制到手机、上传或分享给他人。
